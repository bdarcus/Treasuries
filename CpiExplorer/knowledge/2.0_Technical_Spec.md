# CpiExplorer — Technical Spec

---

## 2.1 Data Sources

### S8: CPI_history.csv
- **R2 URL**: `https://pub-ba11062b177640459f72e0a88d0261ae.r2.dev/bls/CPI_history.csv`
- **Schema**: `Year,Period,PeriodName,NSA,SA` (ascending; oldest row first)
- **Coverage**: January 1913 – present (NSA); January 1947 – present (SA; earlier rows blank)
- **Parsed into**: array of `{ year: number, month: number, date: Date, nsa: number, sa: number }`
  - `month` = BLS period code integer (M01 → 1, M12 → 12)
  - `date` = `new Date(year, month - 1, 1)` (first of month, local time)
  - Skip rows where `Period` does not match `/^M\d{2}$/` (BLS occasionally includes annual averages as `M13`)
  - Rows where SA is blank/`"-"` get `sa: null`

### S3: RefCPI.csv
- **R2 URL**: `https://pub-ba11062b177640459f72e0a88d0261ae.r2.dev/TIPS/RefCPI.csv`
- **Schema**: `date,refCpi` (column header is lowercase; ascending by date)
- **Coverage**: April 1998 – present (daily)
- **Parsed into**: array of `{ date: Date, value: number }`
  - `date` = `localDate(row.date)` (YYYY-MM-DD → `new Date(y, m-1, d)`)

## 2.2 Data Loading (`src/data.js`)

Export two async functions:

```js
fetchCpiHistory()  // → Promise<CpiRow[]>
fetchRefCpi()      // → Promise<RefCpiRow[]>
```

Both fetch with `cache: 'no-store'`. Parse CSV with a simple `parseCsv(text)` helper (split lines, split commas, map headers). Return typed arrays as described in §2.1.

Export types as JSDoc `@typedef` for documentation; no runtime type system.

## 2.3 Calculations (`src/calc.js`)

All functions are pure and synchronous. Each returns `{ labels: string[], values: number[] }` ready for Chart.js. `labels` are ISO date strings (`YYYY-MM-DD`) for monthly data, or `YYYY-MM-DD` for daily Ref CPI.

### calcIndex(rows, field)
Returns the raw index level.
- `rows`: CpiRow[] or RefCpiRow[]
- `field`: `'nsa'` | `'sa'` | `'value'`
- Output: one point per row where `row[field] != null`

### calcYoY(rows, field)
[CPI_Change_YoY](../../knowledge/DATA_DICTIONARY.md#cpi-change-yoy): `(row[t] / row[t-12] − 1) × 100`
- Monthly data: look back exactly 12 rows (guaranteed same month prior year since rows are sorted monthly)
- Daily Ref CPI: look back to same calendar date prior year (binary search by date)
- Drop first 12 months (no prior-year data available)

### calcMoM(rows, field)
[CPI_Change_MoM](../../knowledge/DATA_DICTIONARY.md#cpi-change-mom): `(row[t] / row[t-1] − 1) × 100`
- Monthly: look back 1 row
- Daily: look back to same calendar date prior month
- Drop first row

### calcRolling(rows, field, windowMonths)
[Rolling_CPI_Change](../../knowledge/DATA_DICTIONARY.md#rolling-cpi-change): `(row[t] / row[t-N] − 1) × 100`
- `windowMonths`: integer number of months (12 = 1Y, 60 = 5Y, etc.)
- Monthly: look back `windowMonths` rows
- Daily Ref CPI: approximate as `windowMonths × 30.44` days (calendar lookup)
- Drop first `windowMonths` entries

### calcP2P(rows, field, startDate, endDate)
[CPI_Change_P2P](../../knowledge/DATA_DICTIONARY.md#cpi-change-p2p): single scalar, not a series.
- Find closest row on or before `startDate`; find closest row on or before `endDate`
- Return `{ startVal, endVal, changePct, cagr, months }`
- `months` = integer month count between start and end rows
- [CPI_CAGR](../../knowledge/DATA_DICTIONARY.md#cpi-cagr): `((endVal / startVal)^(12 / months) − 1) × 100`
- Also return the full filtered series (index level) between start and end for the chart

### calcStats(labels, values, mode)
Computes summary stats from the currently rendered series:
- `current`: last value in series
- `start`: first value
- `changePct`: `(current / start − 1) × 100` if mode is Index; last value if mode is % change
- `cagr`: annualized from start to current (months derived from labels)
- `peak`: max value + date
- `trough`: min value + date
Returns a plain object with those fields.

## 2.4 Chart Rendering (`src/chart.js`)

Uses **Chart.js 4.x** (CDN) and **Hammer.js 2.x** (CDN) for touch support. Imports `handleChartKeydown`, `setupAxisWheelZoom`, `snapYBounds`, `snapYAfterZoom` from `../../shared/src/chart-keys.js`.

### createChart(canvasId, config)
Creates a new Chart.js instance:
- **Type**: `'line'`
- **X axis**: `type: 'time'`, `time.unit` auto (Chart.js selects based on range)
- **Y axis**: auto-scaled; `snapYBounds` applied after render
- **Dataset**: single line, no point dots for large series (> 500 points: `pointRadius: 0`), small series: `pointRadius: 3`
- **Plugins**: no legend (single series); custom tooltip showing date + value
- **Interaction**: `mode: 'index'`, `intersect: false`
- Returns the Chart instance

### updateChart(chart, labels, values, yLabel)
Updates an existing chart in place (`chart.data`, `chart.options.scales.y.title`). Calls `chart.update('none')` then `snapYAfterZoom(chart)`.

### resetZoom(chart)
Sets `chart.options.scales.x.min = undefined`, `.max = undefined` and calls `snapYBounds(chart)`.

### Zoom/Pan wiring
In `app.js` (not `chart.js`): attach `keydown` listener to `document` → `handleChartKeydown`. Attach `wheel` listener to canvas → `setupAxisWheelZoom`.

## 2.6 App Orchestration (`src/app.js`)

### State
```js
let cpiRows = null;       // CpiRow[] from fetchCpiHistory()
let refCpiRows = null;    // RefCpiRow[] from fetchRefCpi()
let chart = null;
let state = {
  dataSources: ['cpi-nsa'], // array of 'cpi-nsa' | 'cpi-sa' | 'ref-cpi'
  displayMode: 'index',    // 'index' | 'yoy' | 'mom' | 'rolling' | 'p2p'
  rollingMonths: 12,
  startDate: null,         // Date | null (null = full range)
  endDate: null,
  p2pStart: null,
  p2pEnd: null,
  logScale: false,
};
```

### Initialization sequence
1. Show "Loading…" in status
2. `Promise.all([fetchCpiHistory(), fetchRefCpi()])` — fetch both sources concurrently
3. Store results in `cpiRows`, `refCpiRows`
4. Set default date range inputs to data extents
5. Call `render()`
6. Wire all control event listeners

### render() — called on every control change
1. Determine active rows: `cpiRows` (field `'nsa'` or `'sa'`) or `refCpiRows` (field `'value'`)
2. Filter rows to `[state.startDate, state.endDate]` range (skip for p2p — uses its own inputs)
3. Call the appropriate `calc.*` function based on `state.displayMode`
4. For `p2p`: compute scalar result → display in sidebar; also feed full filtered index series to chart
5. `updateChart(chart, labels, values, yAxisLabel(state))`
6. Compute and render stats strip from `calcStats(labels, values, state.displayMode)`
7. Update status line with last data date

### yAxisLabel(state) → string
- `'index'` → `'CPI Index'`
- `'yoy'` → `'Year-over-Year %'`
- `'mom'` → `'Month-over-Month %'`
- `'rolling'` → `` `Trailing ${state.rollingMonths}m Change %` ``
- `'p2p'` → `'CPI Index'`

### Control event wiring
- `dataSource` radio change → reset `startDate`/`endDate` to source extents → `render()`
- `displayMode` radio change → show/hide rolling/p2p/date-range sections → `render()`
- Rolling preset buttons → set `state.rollingMonths` → `render()`
- Rolling custom input → debounce 400ms → set `state.rollingMonths` → `render()`
- Date range inputs → update `state.startDate`/`endDate` → `render()`
- Full History button → reset to source extents → `render()`
- `keydown` on document → `handleChartKeydown`
- `wheel` on canvas → `setupAxisWheelZoom`

## 2.7 Fetch Script (`scripts/fetchCpiHistory.js`)

**Purpose**: Populate [S8](../../knowledge/DATA_DICTIONARY.md#s8) in R2. Run manually or as a local scheduled task (monthly, on BLS release day).

**BLS API**: `POST https://api.bls.gov/publicAPI/v2/timeseries/data/`
- Series: `["CUUR0000SA0", "CUSR0000SA0"]`
- Free tier limit: 10 years per request → fetch in 10-year batches from 1913 to `currentYear`
- No API key required (50 req/day limit, well within monthly usage)

**Algorithm**:
1. Compute decade start years: `[1913, 1923, 1933, ..., currentDecadeStart]`
2. For each batch: POST `{ seriesid, startyear, endyear }` — `endyear = min(startyear + 9, currentYear)`
3. Collect all `{ year, period, periodName, nsa, sa }` rows; merge by `year+period` key
4. Filter: keep only rows where period matches `/^M\d{2}$/` (drop M13 annual averages)
5. Sort ascending: year ASC, period ASC
6. Serialize as CSV: `Year,Period,PeriodName,NSA,SA`
7. Upload to R2: key `bls/CPI_history.csv` using `uploadToR2` from `./r2.js`

**Dependencies**: `dotenv`, `./r2.js` (same as other scripts in `scripts/`).

**Usage**:
```
node scripts/fetchCpiHistory.js [--write]
```
Without `--write`: prints first 5 rows and total count (dry run). With `--write`: uploads to R2.
