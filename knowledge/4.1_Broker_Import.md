# 4.1 Broker Import

## Dependencies
**Requires:** 2.1 TIPS Basics (for CUSIP validation and face value context)
**Feeds into:** 4.0 TIPS Ladder Rebalancing (provides the `holdings` input)

## Purpose
Allows users to upload raw positions CSV exports from major brokerages (Fidelity, Schwab) instead of manually formatting `cusip,qty` files. The UI intercepts the upload, parses out the accounts, and lets the user select which accounts to merge into the Rebalance engine.

---

## Supported Broker Formats

### 1. Fidelity (`Positions*.csv`)
- **Headers:** Expects columns containing `Account Number`, `Account Name`, `Symbol` (or CUSIP), and `Quantity`.
- **Quantity formatting:** Usually expressed as bond face value (e.g., `10,000`). The parser strips non-numeric characters and divides by 1,000 to convert to bond count.
- **Account grouping:** Uses `Account Name` (or `Account Number` if name is missing).

### 2. Schwab (`*All Accounts*.csv` or single account)
- **Headers:** Expects columns matching `Symbol` and `Qty`. 
- **Multi-account structure:** Schwab exports often omit the Account column in the data rows. Instead, they use header blocks: `"[Account Name] ...[Last 4 Digits]"`.
- **Quantity formatting:** Also exports face value. Converted via `Math.round(faceValue / 1000)`.

---

## Process Flow

1. **File Selection (`index.html`)**: When `#holdings-file` changes, the file is read via the browser `File` API.
2. **Auto-Detection**: The first 20 lines are scanned. If broker-specific keywords are found (e.g., `Account Number`, `Positions for`, `Symbol` + `Quantity`), the file routes to `parseBrokerCSV`.
3. **Extraction & Validation (`src/broker-import.js`)**: 
   - Skips non-TIPS assets by validating symbols against the live `tipsMap` loaded from `TipsYields.csv`.
   - Groups valid TIPS into an `accounts` map.
4. **Selection Modal**: If > 1 account is found, `#account-overlay` appears. The user toggles checkboxes.
5. **Handoff**: The selected accounts are concatenated, merged (summing quantities of identical CUSIPs), and passed into `runRebalance` identically to a standard CSV.
