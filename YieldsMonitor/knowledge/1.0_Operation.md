# 1.0 Yields Monitor Operation

## Purpose
The Yields Monitor provides real-time and historical visualization of Treasury yields (Nominal and TIPS). It focuses on high-resolution intraday movements and daily historical trends, ensuring consistent yield change reporting across all time horizons.

## Data Architecture

### Data Sources
1. **R2 Historical Baseline**: Daily closing yields stored as JSON files (e.g., `US30YTIPS_history.json`). Updated daily via GitHub Actions.
2. **CNBC Real-Time Feed**: High-resolution intraday data.
   - `timeRange=1D`: Used for the **2D** UI range (approx. 2 days of intraday).
   - `timeRange=5D`: Used for the **10D** UI range (approx. 5 days of high-res intraday).
   - **Note**: The 5D feed is the source of truth for the previous day's close (17:00–17:05 ET).

### Data Blending ("Live Tip")
For long-term views (**1Y**, **10Y**, **ALL**), the app fetches historical data from R2 and appends the latest intraday data from CNBC.
- To ensure consistency, the **5D** CNBC feed is used as the "tip" for all views.
- This ensures that the "Latest Yield" and "Change" shown in the sidebar are identical regardless of the chart's active time range.

## Yield Change Calculation

### Logic
The yield change is calculated as the difference between the **Latest Yield** and the **Previous Close**.

1. **Latest Yield**: The most recent data point from the 5D CNBC feed.
2. **Previous Close**:
   - The algorithm searches the 5D data backwards from the current moment.
   - It identifies the first point that belongs to a day *prior* to the latest point's day (using ET wall-clock time).
   - It specifically looks for a point between **17:00 and 17:05 ET** (market close).
   - If a 17:00 point is missing, it falls back to the last available point for that prior day.

### Consistency Requirement
- The Sidebar labels (Yield and Change) must ALWAYS be calculated from the 5D data.
- The Chart for 1Y/10Y may show daily points, but the summary labels must reflect the high-resolution real-time state.

## Timezone Management
- All data is handled as UTC instants but interpreted using the `America/New_York` timezone for:
  - Determining market open (08:00 ET) and close (17:00 ET).
  - Shading after-hours periods in intraday charts.
  - Formatting X-axis ticks and tooltip titles.
