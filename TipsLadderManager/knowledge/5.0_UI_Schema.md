# 6.0 UI Schema

## Dependencies
**Requires:** 4.0 TIPS Ladder Rebalancing, 5.0 Computation Modules

**This file specifies the UI column schema, table rendering logic, and popup routing.**
It is the authoritative reference for `render.js` and `drill.js`.

---

## Purpose

A single `COLS` array drives all table rendering for both Rebalance and Build modes.
Adding, removing, or changing a column requires editing one object in `COLS` only —
headers, cell values, sub-row values, totals, and popup wiring all derive from it.

---

## COLS Schema

Each column is defined by one object:

```js
{
  label,        // string — header text and display label
  key,          // string — colKey used for popup routing and data binding
  fmt,          // 'str' | 'fy' | 'amt' | 'qty' | 'sgn' — formatting type
  rebalOnly,    // bool — column appears in Rebalance mode only (default false)
  buildOnly,    // bool — column appears in Build mode only (default false)
  value,        // fn(d, r) → number|string — main row cell value
  subValue,     // fn(d) → number|string | null — Gap sub-row cell value (bracket rows only)
  total,        // bool — include in totals row (default false)
  drill,        // bool — cell is drillable (clickable popup) (default false)
  drillCond,    // fn(v, d) → bool | null — override drill condition (default: v != null && v !== 0)
}
```

`render.js` filters `COLS` by mode before rendering:
```js
const activeCols = COLS.filter(c => mode === 'rebal' ? !c.buildOnly : !c.rebalOnly);
```

## Sub-Row Logic

For bracket target rows only, a Gap sub-row is rendered showing excess quantities.

**Row ordering:**
- Lower bracket: main row first, Gap sub-row second
- Upper bracket: Gap sub-row first, main row second

This ordering places the gap coverage visually between the two bracket rows:
```
[lower bracket main]
[lower bracket gap]
... [non-bracket years] ...
[upper bracket gap]    ← before upper main
[upper bracket main]
```

**Bracket identification:** `d.excessQtyBefore > 0 || d.excessQtyAfter > 0` (rebalance) or `d.excessQty > 0` (build).
Upper bracket: `d.fundedYear === summary.brackets.upperYear` (rebalance) or `d.fundedYear === summary.upperYear` (build).

**3-bracket mode:** New lower bracket (`summary.newLowerYear`) also gets a Gap sub-row, inserted between orig lower Gap and the non-bracket rebalance years.

**Spec reference:** 4.0 §FY vs Excess (Bracket Years).

## Popup Routing

`drill.js` exports a single function:

```js
buildDrillHTML(d, colKey, summary)
```

Returns HTML string for the popup content. Routes by `colKey`:

| colKey(s) | Popup content | Spec reference |
|---|---|---|
| `amtBefore`, `amtAfter`, `amount` | bondVars → qty → principal + ownCoupon + laterMatInt [+ preLadderCreditForYear if > 0, Build only] | 4.0 Phase 5 ARA Before/After |
| `costBefore`, `costAfter`, `cost` | price → indexRatio → costPerBond → qty → total cost | 2.1 Cost per Bond |
| `qtyAfter`, `qty` | bondVars → piPerBond → DARA → laterMatInt → fyQty (+ excess chain for brackets) | 4.0 Phase 4 step 2 |
| `cashDelta` | costPerBond → qtyDelta → cashDelta | 4.0 Named Quantities Cash Delta |
| `gapAmtBefore`, `gapAmtAfter` | "see Duration Calcs" reference → weight value → totalCost → excessQty → piPerBond → gapAmt | 4.0 Phase 3c, Phase 2 |
| `gapCostBefore`, `gapCostAfter` | "see Duration Calcs" reference → weight value → totalCost → excessQty → costPerBond → gapCost | 4.0 Phase 3c |

**Duration Calcs popup** (`_infoPopups.duration`) is mode-isolated: `_rebalDurationRows` and `_buildDurationRows` are stored separately in `index.html`; the mode-toggle handler restores the correct set. This prevents Build runs from overwriting the 3-bracket popup rows when the user switches back to Rebalance.

**Duration chain** (shown in Duration Calcs popup only; gap cost/amount popups reference it rather than repeat it):
```
Gap avg duration (Phase 2)
  → bracket durations (Phase 3b)
  → bracket weights (Phase 3c / 3c-alt for 3-bracket)
  → targetExcess$ = gapTotalCost × weight
  → excessQty = ROUND(targetExcess$ / costPerBond)
  → amount or cost
```

**Spec reference:** 5.0 §bond-math.js (bondVars rows helper), 4.0 Phase 2-3, 2.1.

## Formatting Types

| fmt | Description | Example |
|---|---|---|
| `str` | Raw string, left-aligned | `912810FR4` |
| `fy` | Funded year integer or 'Gap' | `2036` / `Gap` |
| `amt` | Dollar amount, rounded, thousands-separated | `42,317` |
| `qty` | Integer bond quantity | `12` |
| `sgn` | Signed integer with + prefix | `+3` / `−5` |

## Totals Row

Only columns with `total: true` show values in the totals row. Others show blank.
Total = sum of main row values + sum of sub-row values where applicable.

**Spec reference:** totals are display-only sums, not spec-defined quantities.
