# 5.0 Computation Modules

## Dependencies
**Requires:** 2.1 TIPS Basics, 3.0 TIPS Ladders, 4.0 TIPS Ladder Rebalancing

**This file specifies the module interfaces for shared computation libraries.**
The underlying math is defined in the files above; this file defines function contracts,
input/output shapes, and which spec section each function implements.

---

## Purpose

Shared modules eliminate duplicated formula code across `rebalance-lib.js` and `build-lib.js`.
Each function has a single authoritative implementation. Both orchestrators import from these modules.

Traceability: every function maps to a spec formula. If a computed value is wrong, check the
spec formula first, then the function implementation.

---

## bond-math.js

Pure per-bond calculations. No ladder state, no running pools. Given a bond and refCPI, returns
all derived per-bond quantities. Formulas defined in **2.1 TIPS Basics**.

### `bondCalcs(bond, refCPI)`

**Inputs:**
- `bond` — `{ price, coupon, baseCpi, maturity: Date }`
- `refCPI` — number (settlement date reference CPI)

**Returns:**
```
{
  indexRatio,        // refCPI / bond.baseCpi                        → 2.1 Index Ratio
  principalPerBond,  // 1,000 × indexRatio                           → 2.1 Inflation-Adjusted Principal
  costPerBond,       // price/100 × indexRatio × 1,000               → 2.1 Cost per Bond
  nPeriods,          // 1 if maturity month < 7, else 2              → 2.1 Last-Year Interest (TIPS)
  couponPerPeriod,   // bond.coupon / 2
  ownRungInt,        // principalPerBond × couponPerPeriod × nPeriods → 2.1 Last-Year Interest (TIPS)
  piPerBond,         // principalPerBond + ownRungInt                 → 2.1 P+I per Bond (TIPS)
  annualInt,         // principalPerBond × bond.coupon               → 2.1 Inflation-Adjusted Annual Interest
}
```

**Note:** `piPerBond` here equals `principalPerBond × (1 + coupon/2 × nPeriods)` — algebraically
identical to `adjustedPrincipal + lastYearInterest`. Both forms are valid; see 4.0 Named Quantities.

### `calculateMDuration(settlementDate, maturityDate, coupon, yield)`

Modified duration of a bond. Moved from `rebalance-lib.js` to here.

**Spec reference:** standard bond duration formula (not separately documented — implementation
is the reference). Used in: 4.0 Phase 2 (synthetic gap durations), Phase 3b (bracket durations).

---

### `rungAmount(qty, piPerBond, laterMatInt)`

```
rungAmount = qty × piPerBond + laterMatInt
```

**Spec reference:** 4.0 Phase 5 ARA After formula. Also the structure of `fy_amt` in build-lib.
Ensures the `laterMatInt` term is never omitted in amount calculations.

## gap-math.js

Gap year analysis and bracket sizing. Formulas defined in **4.0 Phase 2 and Phase 3**.

### `interpolateYield(anchorBefore, anchorAfter, targetDate)`

Linear yield interpolation between two anchor bonds.

```
yield = anchorBefore.yield +
        (targetDate - anchorBefore.maturity) × (anchorAfter.yield - anchorBefore.yield) /
        (anchorAfter.maturity - anchorBefore.maturity)
```

**Spec reference:** 4.0 Phase 2, 3.0 Synthetic TIPS Construction.

---

### `syntheticCoupon(yield)`

```
syntheticCoupon = MAX(0.00125, FLOOR(yield × 100 / 0.125) × 0.00125)
```

**Spec reference:** 4.0 Phase 2, 3.0 Synthetic TIPS Construction.

---

### `calcGapParams(gapYears, anchorBefore, anchorAfter, settlementDate, refCPI, dara, prelim)`

For each gap year: interpolate yield → synthetic coupon → synthetic maturity (Feb 15) →
MDURATION → gap qty estimate.

**Returns:**
```
{
  avgDuration,  // mean of synthetic durations across all gap years  → 4.0 Phase 2 Gap Avg Duration
  totalCost,    // sum of (gapQty × $1,000) across all gap years     → 4.0 Phase 2 Gap Total Cost
}
```

**Spec reference:** 4.0 Phase 2.

---

### `bracketWeights(lowerDuration, upperDuration, avgGapDuration)`

```
lowerWeight = (upperDuration - avgGapDuration) / (upperDuration - lowerDuration)
upperWeight = 1 - lowerWeight
```

**Returns:** `{ lowerWeight, upperWeight }`

**Spec reference:** 4.0 Phase 3c.

---

### `bracketWeights3(d1, d2, d3, Dg, origExcess$, gapTotalCost)`

Three-bracket weight solver. `d1` = orig lower duration, `d2` = new lower duration, `d3` = upper duration, `Dg` = gap avg duration.

```
w1     = gapTotalCost > 0 ? origExcess$ / gapTotalCost : 0   // fixed: current orig lower fraction
w2_raw = (Dg - d3 + w1×(d3 - d1)) / (d2 - d3)
w2     = max(0, w2_raw)   // clamp: if orig lower overshoots, skip new lower
w3     = 1 - w1 - w2
```

**Returns:** `{ origLowerWeight: w1, newLowerWeight: w2, upperWeight: w3, feasible: w2raw >= 0 }`

**Infeasibility note:** When `w1 > 1` (orig lower excess alone exceeds gap total cost), `w2 = 0` and `w3 = 1 − w1 < 0`. In this case no new lower or upper bracket bonds are purchased for gap-excess purposes (`bracketExcessTarget` is clamped to `max(0, ...)` in the sweep). Duration match is approximate. The popup shows "Infeasible" instead of the match equation.

**Spec reference:** 4.0 §3-Bracket Mode.

---

### `bracketExcessQtys(totalCost, lowerWeight, upperWeight, lowerCostPerBond, upperCostPerBond)`

```
lowerExQty = ROUND(totalCost × lowerWeight / lowerCostPerBond)
upperExQty = ROUND(totalCost × upperWeight / upperCostPerBond)
```

**Returns:** `{ lowerExQty, upperExQty }`

**Spec reference:** 4.0 Phase 3c (`targetExcess$ per bracket = gapTotalCost × bracketWeight`),
4.0 Named Quantities `excessQtyAfter`.

## build-lib.js — Pre-Ladder Interest Option

**Option:** `preLadderInterest` (boolean, default `false`)

When `firstYear > settlementDate.getFullYear()` and the option is enabled, coupon payments
received from all ladder bonds BEFORE the ladder starts is credited against the earliest
funded years, potentially eliminating the need for TIPS in those years.

### Algorithm

**Step 1 — Prelim sweep** (unchanged, long→short): produces `prelim[year].annualInterest`
and `prelim[year].laterMatInt` for all funded years.

**Step 2 — Compute pre-ladder pool**
```
preLadderYears = max(0, firstYear - settlementDate.getFullYear())
totalAnnualInt = Σ prelim[year].annualInterest  (all years in ladder)
preLadderPool  = preLadderYears × totalAnnualInt
```
The pool represents total coupon payments from all ladder bonds received in the calendar years
[settlementYear, firstYear−1]. It is computed from prelim quantities — the approximation is
slightly conservative (pool may be modestly over-estimated if some years are zeroed).

**Step 3 — Apply pool (short→long):** determine `zeroedYears`
```
remaining = preLadderPool
for year in rangeYears ascending (short→long):
  need = DARA - prelim[year].laterMatInt    // what own bonds must cover
  if remaining >= need:
    zeroedYears.add(year)
    remaining -= need
  else:
    break   // pool exhausted; this year gets partial credit

  partialCredit = remaining (for the first non-zeroed year, if pool wasn't exhausted)
```

**Step 4 — Final output:** for each year:
- `fundedYearQty = 0` if year ∈ zeroedYears
- `fundedYearQty = max(0, ROUND((DARA - laterMatInt - partialCredit) / pi))` for the first non-zeroed year
- Normal `fyQty(DARA, laterMatInt, pi)` for all subsequent years

Bracket `excessQty` is computed normally and applied on top regardless of zeroing —
duration matching still applies to bracket years even if their funded-year qty is zero.

**Summary fields added when `preLadderInterest` enabled:**
- `preLadderYears` — calendar years before ladder start
- `preLadderPool` — total pre-ladder coupon pool ($)
- `zeroedFundedYears` — array of years with fundedYearQty = 0

## ladder-math.js

Rung-level sweep helpers. Formulas defined in **4.0 Phase 4**.

### `fyQty(dara, laterMatInt, piPerBond)`

```
fyQty = MAX(0, ROUND((dara - laterMatInt) / piPerBond))
```

**Spec reference:** 4.0 Phase 4, step 2 `targetFYQty`.

---

### `laterMatIntContribution(qty, bond, refCPI)`

```
contribution = qty × 1,000 × indexRatio × bond.coupon
             = qty × annualInt   (where annualInt from bondCalcs)
```

What a processed rung adds to the running `laterMatInt` pool.

**Spec reference:** 4.0 Phase 4, step 4.

## Variable Naming Harmonization

The refactor standardizes naming across both libs (divergence noted in 4.0 §Naming divergence note):

| Concept | Old rebalance-lib | Old build-lib | Harmonized |
|---|---|---|---|
| Lower bracket duration | `lowerDuration` | `lowerDur` | `lowerDuration` |
| Upper bracket duration | `upperDuration` | `upperDur` | `upperDuration` |
| Lower bracket weight | `lowerWeight` | `lowerWt` | `lowerWeight` |
| Upper bracket weight | `upperWeight` | `upperWt` | `upperWeight` |
| FY Qty | `fyQty` | `fy_qty` | `fyQty` |
| Cost per Bond | `costPerBond` | `fy_costPerBond` | `costPerBond` |
| Excess Qty | `excessQtyAfter` | `ex_qty` | `excessQty` |
